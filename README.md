# ATown

ATown 是一个俯视视角的中国现代小镇生活模拟器。玩家以新居民身份下车进镇，在学校、医院、派出所、居民小区、镇政府、银行、市场、饭馆、药房、邮政所、招待所、客运站和公园之间生活与走访。

镇上总有一个现实项目正在酝酿，例如重办庙会、修缮危房或让老供销社重新开门。但游戏没有任务列表、地图标点、支持度和进度条。玩家只能靠走访和自由对话了解每个人的立场：封街可能影响面馆进货，锣鼓可能打扰老人，修房可能堵住消防通道，新店也可能伤害旧店生意。事情能否办成，取决于玩家是否听懂这些具体难处，并提出让当事人愿意亲自参加协商的办法。

## 当前实现

- 进入游戏前可选择离线模式或 AI 模式
- 36×26 俯视城镇地图、居民公园和 12 个独立建筑室内场景
- 每局至少 26 名可交谈居民，覆盖教师、医生、民警、商户、学生、儿童、家长、老人、司机等身份
- 街道和公园常驻多名居民，并有轿车、快递三轮车、公交和儿童设施
- 每名 NPC 均有稳定的 `agentId`、独立人物设定和独立对话历史
- 方向键或 WASD 逐格移动、碰撞、点击寻路
- 走到机构门口自动进入，室内可与当值 NPC 和生活物件互动
- 靠近 NPC 后按 `E` 或点击人物开始自由对话
- 靠近水井、长凳、灶台、扫帚、木料、竹榻或收音机后按 `F` 生活互动
- 饱腹、精力、心情和零钱会随走路、吃饭、劳动、休息及时间变化
- 五位镇民拥有不同公开态度、私下顾虑和可协商条件
- 后端隐藏所有项目数值，只通过对话、人物行为和镇上变化呈现结果
- 没有任务列表、显式目标、进度条或预设对话选项
- 桌面键盘、鼠标和移动端方向按钮支持

## 本地运行

需要 Node.js 18+、npm 和 Go 1.21+。

后端：

```bash
cd backend
go mod download
go run ./cmd
```

前端：

```bash
cd frontend
npm install
npm run dev
```

Windows PowerShell 如果阻止执行 `npm.ps1`，可改用 `npm.cmd install` 和 `npm.cmd run dev`。

访问 `http://localhost:5173`。开发环境下浏览器会直接连接当前主机的 `8080` WebSocket 端口，避免 Vite 热更新中断游戏连接时产生代理错误。

生产环境如果前后端不在同一域名，可在构建前设置：

```bash
VITE_WS_URL=wss://example.com/ws
```

## 运行模式

### 离线模式

不需要模型配置。后端从固定的现实题材模板中随机选择学校撤并、夜市治理或医院急救通道等剧情，并使用规则化角色对话。该模式适合本地开发、自动测试和无网络演示。

### AI 模式

进入前由用户填写：

- OpenAI 兼容 Base URL，例如 `https://api.openai.com/v1`
- 模型名称，例如 `gpt-4o-mini`
- API Key
- 城镇生成超时，默认 300 秒，可设置 60 至 900 秒
- NPC 对话超时，默认 120 秒，可设置 30 至 300 秒

后端调用 `{baseUrl}/chat/completions`。模型会现场生成不同的镇名、机构名称、16 至 20 名居民、人物关系和公共剧情。每个 NPC 后续通过自己的 Agent 上下文进行对话，拥有独立身份、顾虑和最近 10 轮记忆。

API Key 只保存在当前 WebSocket 会话的内存中，不写入浏览器存储、城镇状态或磁盘。模型生成或对话失败时，界面会显示真实错误，不会静默切换为模板回答。

生成完整城镇需要模型一次返回较大的结构化 JSON。部分中转接口首个响应头会超过 90 秒，因此城镇生成和 NPC 对话使用独立超时。若出现“模型在 300 秒内没有完成响应”，说明接口地址可达但所选模型或中转队列过慢，可以把生成超时提高到 600 秒，或换用响应更快且支持 JSON 输出的模型。

AI 生成结果会经过后端校验和归一化：学校、医院、公安、法院等必需机构始终存在；NPC 不足时补齐离线居民；重复 Agent ID 会自动修正；剧情入口人物必须是实际存在的 NPC。

## 剧情约束

剧情状态对前端隐藏，但后端按四阶段推进：

1. 听见风声：NPC 只透露与自己有关的有限信息。
2. 厘清分歧：走访多名利益相关者后，NPC 才会坦白真实顾虑。
3. 协调方案：玩家必须回应具体条件，不能靠空泛承诺获得支持。
4. 落地协商：讨论书面承诺、责任分工和执行安排。

AI Agent 的系统提示包含当前阶段约束，不能泄露全局答案、其他 NPC 秘密或隐藏支持度。

## 如何游玩

1. 用方向键、WASD 或点击地图在镇上行走。
2. 走到建筑门口会直接进入，室内下方的门可以回到街上。
3. 靠近镇民，按 `E` 或点击人物搭话。
4. 靠近生活物件按 `F`，可以吃面、打水、休息、听广播或帮镇民干活。
5. 不要直接问“你支持吗”。先问对方为什么犹豫、最担心什么。
6. 找其他人核实情况，再把具体解决办法带回来。
7. 当各方愿意坐下来自己表达条件，镇上的环境会自然出现变化。

## 时间与住宿

- 所有模式采用唯一且固定的时间比例：现实 1 秒等于虚拟世界 1 分钟，因此现实 1 分钟等于虚拟世界 1 小时。
- 移动、对话、吃饭和休息不会额外跳过时间；搬一次货固定消耗 30 分钟。
- 每天 `21:50` 准时进入夜间休息结算，第二天从 `07:00` 开始。
- 玩家需先在招待所前台办理住所，18:00 后可回到前台提前入住。
- 18:00 仍未办理住所时系统会提醒；21:50 前没有入住则按露宿街头结算。
- 未办理入住会在客运站屋檐下流落街头，次日精力降至很低，心情也会下降。
- 房间只保留一晚，次日自动退房。

招待所房价：

| 房型 | 价格 | 次日精力 |
| --- | ---: | ---: |
| 五人间 | 15 元 / 晚 | 72 |
| 三人间 | 28 元 / 晚 | 84 |
| 单人间 | 45 元 / 晚 | 100 |

游戏内货币统一使用“元”。初始携带 60 元，吃饭、住宿和临时劳动都会改变余额。

每局项目不同，没有固定的演示台词。有效的交谈通常包含“你担心什么”“会不会影响送货”“能否保留旧柜台”“九点前收声”“先问住户意见”等具体问题和方案。

## 架构原则

前端使用 React、Vite 和原生 WebSocket。地图由后端生成的瓦片数据渲染，前端只发送逐格移动、点击目标和对话请求。

后端使用 Go 与 gorilla/websocket，每个连接拥有独立的内存小镇。移动碰撞、建筑切换和隐式项目状态全部由后端裁决。NPC 的核心信息分为可见状态和隐藏顾虑，隐藏项目及支持状态不会序列化给前端。

当前对话层包含无 API Key 可运行的本地协商引擎。接入 LLM 时，应保留相同原则：只向模型提供当前 NPC 的立场、顾虑、记忆和相关镇情，不得向前端暴露全局解法或任务进度。
